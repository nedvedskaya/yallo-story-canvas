
# Решение для «точь-в-точь» экспорта в лучшем качестве

## Что реально ломает экспорт сейчас
1. `DownloadModal.tsx`: захват идёт с маленького превью + `scale` ограничен `4`, поэтому часть форматов потом фактически апскейлится → мыло/смещения.
2. Видео без текста: при сбое `MediaRecorder` код молча падает в fallback и кладёт «оригинальный файл видео» (без наложенного текста).
3. Экспорт и превью рендерятся разными путями, поэтому неизбежно появляются расхождения.

**Do I know what the issue is?** Да: главная причина — не единый рендер-источник + fallback видео без overlay.

## Предлагаемый подход (рабочий и стабильный)
### 1) Один источник рендера для экрана и экспорта
- Вынести JSX слайда в общий компонент `SlideFrame` (новый файл).
- `SlideCarousel` использует `SlideFrame` для предпросмотра.
- `DownloadModal` использует **тот же `SlideFrame`** в скрытом offscreen-контейнере, но в **нативном размере** (1080x1350, 1080x1920 и т.д.).
- Для PNG/PDF захват в native-size с `scale: 1` (без апскейла маленького превью).

### 2) Экспорт PNG/PDF в максимальном качестве
- Убрать cap `scale=4` и уйти от захвата мобильного превью вообще.
- Перед capture: `document.fonts.ready`, `img.decode()`, стабилизация layout (2 `requestAnimationFrame`).
- Для PDF вставлять изображения 1:1 по px-формату страницы (без дополнительного ресайза).

### 3) Видео-экспорт с текстом поверх (как на сайте)
- Переписать video export так, чтобы overlay рисовался всегда из content-layer `SlideFrame`.
- Добавить корректный выбор MIME:
  - `video/mp4;codecs=h264,aac` (если поддерживается),
  - иначе `video/webm;codecs=vp9/vp8`.
- Убрать тихий fallback на «голое видео без текста».
- Если на устройстве запись с overlay невозможна — показывать явный toast + альтернативу (статичный кадр/исходник), чтобы не было «кажется сохранилось правильно».

### 4) «Всё в одном месте»
- Один ZIP:
  - статичные слайды: `slide-N.png`
  - видео-слайды: `slide-N.mp4` или `slide-N.webm`
- Внутри `export-report.json` (формат, codec, ошибки/фоллбеки) — чтобы диагностировать проблемы на конкретных телефонах.

## Файлы, которые нужно изменить
1. `src/components/editor/SlideFrame.tsx` (новый) — единый рендер слайда.
2. `src/components/editor/SlideCarousel.tsx` — заменить локальный JSX на `SlideFrame`.
3. `src/components/editor/DownloadModal.tsx` — offscreen native export, новый video pipeline, явные ошибки вместо silent fallback.
4. (опц.) `src/components/editor/export-utils.ts` — утилиты ожидания ассетов, MIME negotiation, prepareCapture.

## Технические детали (важно)
- В браузере нельзя гарантировать идеальные 100% на всех iOS/Android из-за ограничений `MediaRecorder/html2canvas`.
- Для **абсолютно 1:1 без компромиссов** (особенно видео+текст+аудио) нужен cloud-render (Chromium + FFmpeg).  
- Но описанный выше рефактор уже уберёт текущие критичные артефакты и даст максимально близкий к экрану результат в клиентском режиме.

## План проверки после внедрения
1. Сравнение «экран vs PNG» на 4 форматах (carousel/square/stories/presentation).
2. Проверка PDF: переносы, выделения, позиция текста, отсутствие растяжения.
3. Проверка video export: есть ли overlay-текст + звук на iOS и Android.
4. Проверка ZIP «всё вместе»: корректные имена, форматы, отсутствие silent fallback.
